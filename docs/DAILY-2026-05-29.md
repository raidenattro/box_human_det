## 三、RTSP 采帧与推理可靠性

- **`LatestFrameBuffer` TTL**（`RTSP_FRAME_BUFFER_TTL_SEC`，默认 1s）：超时清空缓冲，infer 不再对断流旧帧反复出 pose/碰撞。
- **`rtsp_capture` 自动重连**：读帧连续失败达阈值（`RTSP_RECONNECT_FAIL_STREAK`，默认 90）后重建 `VideoCapture`，避免 infer 进程 RTSP 僵死却不写 Redis。
- infer 容器环境透传 `RTSP_FRAME_BUFFER_TTL_SEC`（`inference_container_service.py`）。
- 现场验证：停推流后 pose snapshot 在约 1s 内停更、event-worker 无新回调；恢复推流后需重连 infer 或依赖自动重连。

## 四、拓扑 Pose 新鲜度（排查「有画面无骨架」）

- 从 Redis 读 `pose:snapshot` + `pose:stream` 最近帧，检测：
  - **`INFER_POSE_FROZEN`**：MTX 有流、infer 在跑，但连续帧 `frame_idx` 增而 `persons` 不变（缓冲旧帧）。
  - **`POSE_STALE`**：infer 在跑但 pose 快照超过 `TOPOLOGY_POSE_STALE_SEC`（默认 3s）未更新。
- `infer→redis` 边 health：ok / warn / error（不再固定 unknown）。
- 抽屉新增 **「Pose / 事件」**：pose 年龄、frame_idx、是否冻结、event-worker 等。
- 根因说明：Monitor 画面走 MTX WHEP；骨架走 infer→Redis→SSE，两路独立。
